# 第 10 章：实战案例 - OpenClaw 自动创建飞书文档

> 从零开始，让 OpenClaw 自动在飞书指定目录创建文档

---

## 10.1 项目背景

在日常工作中，我们经常需要在飞书中创建文档。手动操作既耗时又容易遗漏。

**需求**：
- ✅ 自动在飞书指定目录创建文档
- ✅ 自动写入内容
- ✅ 无需手动操作
- ✅ 可重复使用

**解决方案**：使用 OpenClaw 的飞书技能自动创建文档

---

## 10.2 完整流程

### 10.2.1 第一步：接入飞书

如果你还没有接入飞书，需要先完成飞书机器人的创建和配置。

**前置条件**：
- 飞书账号
- 飞书开放平台开发者权限
- OpenClaw 已安装并运行

**参考章节**：第 1 章 - 接入飞书机器人

### 10.2.2 第二步：直接对话创建

最简单的方式是直接跟 OpenClaw 对话：

```
你：帮我在飞书指定目录创建文档

AI：好的，我来帮你创建飞书文档...
```

**预期结果**：
- OpenClaw 会自动调用飞书 API
- 在指定目录创建新文档
- 返回文档链接

### 10.2.3 第三步：权限配置

在创建文档过程中，可能会遇到权限问题。

**常见问题 1：需要飞书凭证**

```
AI：需要飞书 App ID 和 App Secret
```

**解决方法**：
1. 打开飞书开放平台
2. 进入应用管理
3. 获取 App ID 和 App Secret
4. 配置到 OpenClaw

**常见问题 2：拿不到 AppSecret**

```
AI：我拿不到 AppSecret，这是权限限制
```

**解决方法**：
- 手动填写 AppSecret
- 在 OpenClaw 配置文件中添加权限

**配置文件示例** (`openclaw.json`)：

```json
{
  "feishu": {
    "app_id": "cli_xxxxxxxxxx",
    "app_secret": "xxxxxxxxxxxxxxxx",
    "perm": true
  }
}
```

## 10.3 常见问题和解决方案

### 10.3.1 问题 1：需要重复问两遍

**现象**：
- 第一次不执行
- 第二次才执行

**可能原因**：
- 飞书 API 响应问题
- 模型理解问题
- OpenClaw 调度问题

**解决方案**：
- 多试几次
- 换用不同的 AI 模型
- 检查网络连接

### 10.3.2 问题 2：文档创建成功但内容写不进去

**现象**：
- 文档成功创建
- 但内容是空的

**可能原因**：
- 飞书 API 格式严格
- 模型生成的内容格式不对
- 权限不足

**解决方案**：
- 检查文档格式要求
- 换用不同的 AI 模型
- 检查写入权限

### 10.3.3 问题 3：文档所有权问题

**现象**：
- 创建的文档无法编辑
- 文档所有权归 OpenClaw 而不是用户

**解决方案**：
- 在 `openclaw.json` 中配置飞书权限
- 设置 `perm` 为 `true`

```json
{
  "feishu": {
    "perm": true
  }
}
```

## 10.4 模型选择

### 10.4.1 不同模型的表现

**Qwen3.5-plus**：
- ✅ 优点：理解能力强
- ❌ 缺点：有时候需要重复问两遍
- ❌ 缺点：遇到问题容易"胡说八道"

**智谱模型**：
- ✅ 优点：一次成功
- ✅ 优点：自动修复错误
- ✅ 优点：不胡扯

**建议**：
- 如果遇到模型问题，尝试换一个模型
- 不要迷信某一个模型
- 不同模型的能力差异非常大

### 10.4.2 切换模型

在 OpenClaw 中切换模型：

```
你：切换到智谱模型

AI：好的，已切换到智谱模型
```

## 10.6 经验总结

### 10.6.1 三条经验

**经验 1：不要迷信某一个模型**

- 不同模型的能力差异非常大
- 有时候换一个模型，问题就直接解决了
- 多尝试几个模型

**经验 2：OpenClaw 需要不断调教**

- 你跟它对话越多，它越懂你要干什么
- 自动化能力也会越来越强
- 耐心一点，多试几次

**经验 3：现在还在高速进化期**

- OpenClaw 更新速度非常快
- 很多 bug 是正常的
- 但只要你愿意多试几次，大部分问题都能解决

### 10.6.2 避坑指南

**坑 1：权限配置**
- 确保飞书机器人权限配置正确
- 特别是 `perm` 权限要开启

**坑 2：模型选择**
- 如果遇到问题，先尝试换模型
- 智谱模型表现较好

**坑 3：文档格式**
- 飞书 API 对格式要求严格
- 使用 Markdown 格式
- 避免特殊字符

**坑 4：网络问题**
- 确保网络畅通
- 飞书 API 有时响应慢
- 多试几次
